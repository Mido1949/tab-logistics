Phase 2: قاعدة البيانات

- الهدف: بناء طبقة قاعدة بيانات موثوقة تدعم نشاطات الشحن والتخزين والطلبات. نستخدم PostgreSQL مع Prisma كORM لضمان قابلية التوسع وسهولة الصيانة.
- ما تم اعتماده افتراضيًا:
  - PostgreSQL كقاعدة البيانات
  - Prisma ORM مع Node/TypeScript كمحرك تطبيق
  - نموذج بيانات مبدئي يغطي: المستخدمين، المستودعات، المنتجات، المخزون، العملاء، الطلبات، عناصر الطلب، الشحنات، الشركات الناقلة، والحالات المرتبطة
- ما هي الملفات المضافة:
  - prisma/schema.prisma: نموذج البيانات و علاقاتها
  - prisma/.env.example: قالب لـ DATABASE_URL
  - prisma/seed.js: سكريبت تهيئة بيانات ابتدائية
  - README_PHASE2.md: توثيق خطوة بخطوة
- كيف تعمل خطوة البدء:
 1) اضبط DATABASE_URL في ملف .env (مثال في prisma/.env.example)
 2) تثبيت الحزم: npm install
 3) générer Prisma العميل: npx prisma generate
4) تطبيق الهجرة الأولية: npx prisma migrate dev --name init
5) تشغيل السكريبت للتهيئة: node prisma/seed.js
- مخرجات المرحلة:
  - قاعدة بيانات ذات هيكل مرتب يدعم متطلبات الشحن والتخزين
  - بيانات نموذجية للاختبار والتطوير
- ملاحظات:
  - يمكن إضافة جداول إضافية حسب احتياجات العمل، مثل فواتير، مدفوعات، مسارات الشحن، أو سياسات الخصوصية
- الأسئلة المقترحة للخطوة القادمة:
  1) هل تريد دعم أكثر من عملاء عالميين بمناطق/أسواق متعددة؟
  2) هل نضيف دعم للعناوين المحالة إلى Customer كـ Address سجل مع جيومتري عالية الدقة؟
  3) هل ندمج مع مزود استضافة PostgreSQL معين (DBaaS مثل RDS/Supabase)؟
